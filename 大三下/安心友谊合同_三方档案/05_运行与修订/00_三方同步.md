# 三方同步

这份说明把飞书、本地档案和 GitHub 仓库对齐到同一套组件结构；顶层以飞书原始组件为准，额外的本地支持能力统一收在 05_运行与修订。

## 组件映射

- 合同正文：飞书根节点《安心友谊合同》；本地 00_合同正文；GitHub 仓库保留镜像文本。
- 规划同步：飞书《规划同步》；本地 01_规划同步；GitHub 仓库存月份归档、规划输入和背景材料。
- 双方红线：飞书《双方红线文档》；本地 02_双方红线；GitHub 仓库保留镜像文本。
- 经验分享：飞书《职业or生活技能上的相互教学 + 对 xx 主题的经验分享》；本地 03_经验分享；GitHub 仓库保留镜像文本。
- 聊天记录：飞书《聊天记录》；本地 04_聊天记录；GitHub 仓库保留信件、访谈与转写、会议入口镜像。
- 运行与修订：飞书没有这一层；本地 05_运行与修订；GitHub 仓库承接三方同步、结构修订和复盘输入。

## 本地支持层

- 01_规划同步/90_规划输入：放个人规划和阶段规划输入，不单独升成顶层组件。
- 01_规划同步/99_背景材料：放解释阶段变化所需的背景日志和补充材料。
- 05_运行与修订/03_复盘输入：放阶段复盘和评测输入，不再单独保留顶层“复盘”。

## GitHub 仓库策略

- 默认建立私有仓库。
- 默认只推送这套档案目录，不推送整个 Diary。
- 密码、恢复码和其他凭据不纳入这套档案，也不进入远端。
- 如果 GitHub 未登录，本地先保留可直接推送的 git 目录结构，等登录后补 remote。

- 飞书凭据从环境变量 `FEISHU_APP_ID` 与 `FEISHU_APP_SECRET` 读取，不写入仓库。
